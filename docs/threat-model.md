# Threat model

## Assets and boundaries

The most sensitive assets are active interview text, approved profiles, email associations, authentication material, database/application secrets, and staff search activity. Boundaries are the public internet/CloudFront, browser/API, Lambda/AWS services, staff/Cognito, deployment/GitHub OIDC, and backup restoration.

| STRIDE                  | Threat                                                                     | Primary controls                                                                                                                                                                                             | Residual risk / operator action                                                                                         |
| ----------------------- | -------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------- |
| Spoofing, replay        | Magic-link theft, scanners, token leakage and stolen email                 | URL fragment, POST redemption, ≥256-bit token, HMAC-only storage, single use, purpose/context binding, short expiry, no-referrer                                                                             | Mailbox compromise still grants mailbox authority; shared-mailbox warning is explicit                                   |
| Disclosure              | Account/profile enumeration                                                | Neutral response, mailbox-scoped flow, hashed abuse signals                                                                                                                                                  | Timing and email-delivery side channels require monitoring                                                              |
| Spoofing, tampering     | Session fixation/theft and CSRF                                            | Secure host-only HttpOnly cookies in production, strict Origin, CSRF, session rotation, fixed 30-day absolute expiry                                                                                         | Longer member sessions increase compromised-device exposure; revoke during incidents                                    |
| Elevation               | IDOR or broken profile authorization                                       | Person UUID comes only from server session, ownership predicates, separate integrations/routes, permission tests                                                                                             | Every new route needs positive and negative authorization tests                                                         |
| Elevation, repudiation  | Staff role escalation, compromised session or Cognito misconfiguration     | Cognito password verification, mandatory TOTP, verified issuer/audience/token use, exact groups, permission union, lower-group-only admin page, opaque sessions, audit                                       | Bootstrap administrators and Cognito group mappings remain high trust; review quarterly                                 |
| Spoofing, disclosure    | Staff credential interception, challenge tampering, or account enumeration | HTTPS same-origin form, strict Origin checks, confidential server-side Cognito client, authenticated-encrypted challenge transaction, 10–15 minute expiry, neutral reset response, rate limits, no body logs | The staff Lambda handles credentials transiently; Cognito threat protection and operator account review remain required |
| Tampering               | Prompt injection from member prose or staff query                          | Inputs are delimited as data, candidate-only reranking, schema validation, exact-substring evidence, deterministic fallback                                                                                  | Model output may fail closed and reduce quality                                                                         |
| Tampering, disclosure   | AI hallucinated qualifications or protected/sensitive inferences           | Exact approved prose is primary evidence, unknown IDs/evidence rejected, self-reported cautions, fresh approval after safety revision                                                                        | Free text/model behavior cannot be perfectly classified                                                                 |
| Disclosure              | Sensitive oversharing                                                      | Coaching prompt, deterministic secret/identifier detection, Bedrock Guardrail, fresh review                                                                                                                  | Member review and production logging review remain essential                                                            |
| Disclosure              | Pending conversation exposed outside its member or retained too long       | Person-scoped authorization, encrypted Aurora, no staff route/search use, fixed expiry, approval/deletion cascade, daily cleanup                                                                             | Encrypted backups retain inaccessible copies until rotation; access roles require audit                                 |
| Disclosure              | Conversation retention through telemetry or Bedrock logging                | No body logs/tracing payloads, no CloudFront/API body logs, manual + CLI production preflight                                                                                                                | AWS policy/configuration can change; repeat before deploy and quarterly                                                 |
| Tampering               | SQL injection through search/profile/model                                 | Static parameterized SQL, Zod bounds, planner cannot produce SQL, model has no tools                                                                                                                         | Migration SQL is trusted reviewed repository code                                                                       |
| Disclosure              | XSS through approved prose                                                 | React text rendering, no raw HTML API, CSP, inert-content E2E test                                                                                                                                           | Future rich text would require a new design/review                                                                      |
| Disclosure              | Public execute-api bypass/exfiltration                                     | No public directory, separate Lambda IAM, CloudFront origin header, app authorization                                                                                                                        | Origin secret is defense in depth and must be rotated if exposed                                                        |
| Repudiation, disclosure | Backup restore reintroduces deleted profiles                               | Transactional purge, HMAC purge journal, blocked restore, pseudonym replay before cutover                                                                                                                    | Backups last up to 35 days; fail closed if journal cannot reconcile                                                     |
| Denial of service       | SES bounce/complaint abuse or magic-link spam                              | SES suppression/feedback, deliverability state, WAF/app limits, DLQ/alarms                                                                                                                                   | Distributed abuse may require temporary route/send shutdown                                                             |
| Tampering, disclosure   | Terraform state exposure                                                   | Environment-specific OIDC, exact state prefixes, S3 public block/versioning/KMS/TLS, sensitive outputs                                                                                                       | Deploy roles can read secrets in state; protect environments and audit CloudTrail                                       |
| Tampering, elevation    | GitHub Actions/dependency supply-chain compromise                          | Lockfile/release-age policy, immutable action SHAs, fork plan exclusion, read-only plan state role, Dependabot, CodeQL, CODEOWNERS                                                                           | Maintainers must inspect action SHA, install scripts, lockfile and environment requests                                 |
| Elevation               | Overprivileged Lambda or migration credentials                             | Per-function IAM, app Data API secret only, migration/master ARNs only on migration Lambda, scoped Bedrock/SES/Cognito/SQS                                                                                   | Re-audit IAM whenever a route or worker is added                                                                        |
| Spoofing, disclosure    | Custom-domain cutover sends links or cookies to the wrong host             | Single public base URL, Terraform-managed ACM/alias, strict Origin validation, host-only cookies, cutover runbook                                                                                            | DNS/cache overlap needs a rollback window and smoke test                                                                |
| Denial of service       | Cloud cost/resource exhaustion or Aurora resume storm                      | WAF/app limits, capped Lambda concurrency, 0–2 ACU, bounded resume retry, queues/DLQs, budget                                                                                                                | IP limits do not stop distributed abuse; use incident runbook                                                           |

## Abuse cases explicitly rejected

The product has no member directory, public search, fundraising/marketing export, automated volunteer assignment, compatibility/protected-class scoring, credential approval, background checking, pastoral records, or spiritual-value ranking. Search returns only active, recently reconfirmed approved prose and evidence supported by that prose.

Review this model whenever a data store, model, identity provider, logging sink, public endpoint, custom domain, or staff action is added.
